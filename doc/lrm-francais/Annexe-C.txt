
                   C. Environnement linguistique prédéfini   

Cette annexe décrit la spécification du package STANDARD contenant
tous les identifiants prédéfinis dans la langue. Le corps du paquet correspondant
est défini par l'implémentation et n'est pas affiché.

Les opérateurs prédéfinis pour les types déclarés dans le package
STANDARD sont donnés en commentaires puisqu'ils sont implicitement déclarés. Italique
sont utilisés pour les pseudo-noms de types anonymes (tels que universal_real) et
pour les informations non définies (telles que mise en œuvre_définie et
any_fixed_point_type).

Certains aspects des entités prédéfinies ne peuvent être complètement décrits
dans la langue elle-même. Par exemple, bien que le type d'énumération BOOLEAN
peut être écrit en montrant les deux littéraux d'énumération FALSE et TRUE, le
les formes de contrôle des courts-circuits ne peuvent pas être exprimées dans la langue.

Note:

La définition du langage prédéfinit les unités de bibliothèque suivantes :

   - Le package CALENDRIER (voir 9.6)

   - Le package SYSTEM (voir 13.7)
   - Le package MACHINE_CODE (si fourni) (voir 13.8)
   - La procédure générique UNCHECKED_DEALLOCATION (voir 13.10.1)
   - La fonction générique UNCHECKED_CONVERSION (voir 13.10.2)

   - Le package générique SEQUENTIAL_IO (voir 14.2.3)
   - Le package générique DIRECT_IO (voir 14.2.5)
   - Le package TEXT_IO (voir 14.3.10) 
   - Le package IO_EXCEPTIONS (voir 14.5)
   - Le package LOW_LEVEL_IO (voir 14.6)
