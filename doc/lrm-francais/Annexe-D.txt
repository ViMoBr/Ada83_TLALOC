
                               D. Glossaire  

Cette annexe est informative et ne fait pas partie de la définition standard de
le langage de programmation Ada. Termes en italique dans l'abrégé
les descriptions ci-dessous contiennent elles-mêmes des entrées de glossaire ou sont décrites
dans les entrées pour les termes associés.

Acceptez la déclaration. Voir entrée.

Type d'accès. Une valeur d'un type d'accès (une valeur d'accès) est soit une valeur nulle
valeur, ou une valeur qui désigne un objet créé par un allocateur. Le
l'objet désigné peut être lu et mis à jour via la valeur d'accès. Le
la définition d'un type d'accès précise le type des objets désignés
par les valeurs du type d'accès. Voir aussi collection.

Paramètre réel. Voir paramètre.

Agrégat. L'évaluation d'un agrégat donne la valeur d'un composite
taper. La valeur est spécifiée en donnant la valeur de chacun des
Composants. Une association de position ou une association nommée peut être utilisée
pour indiquer quelle valeur est associée à quel composant.

Allocateur. L'évaluation d'un allocateur crée un objet et renvoie un
nouvelle valeur d'accès qui désigne l'objet.

Type de tableau. Une valeur d'un type tableau est constituée de composants qui sont tous
du même sous-type (et donc du même type). Chaque composant est
distingué de manière unique par un index (pour un tableau unidimensionnel) ou par un
séquence d'indices (pour un tableau multidimensionnel). Chaque index doit être un
valeur d'un type discret et doit se situer dans la plage d'index correcte.

Affectation. L'affectation est l'opération qui remplace la valeur actuelle de
une variable par une nouvelle valeur. Une instruction d'affectation spécifie une variable sur
à gauche, et à droite, une expression dont la valeur est d'être la nouvelle
valeur de la variable.

Attribut. L'évaluation d'un attribut donne un résultat prédéfini  
caractéristique d'une entité nommée ; certains attributs sont des fonctions.

Instruction de bloc. Une instruction de bloc est une instruction unique pouvant contenir
une séquence de déclarations. Il peut également inclure une partie déclarative, et
gestionnaires d'exceptions ; leurs effets sont locaux à l'instruction block.

Corps. Un corps définit l'exécution d'un sous-programme, d'un package ou d'une tâche. UN
Le stub du corps est une forme de corps qui indique que cette exécution est définie
dans une sous-unité compilée séparément.   

Collection. Une collection est l'ensemble des objets créés par
évaluation des allocateurs pour un type d'accès.

Unité de compilation. Une unité de compilation est la déclaration ou le corps d'un
unité de programme, présentée pour compilation sous forme de texte indépendant. C'est
éventuellement précédé d'une clause de contexte, nommant d'autres unités de compilation
dont il dépend moyennant une plus avec des clauses.

Composant. Un composant est une valeur qui fait partie d'une valeur plus grande, ou un
objet qui fait partie d’un objet plus grand.                                              

Type composite. Un type composite est un type dont les valeurs ont des composants.
Il existe deux types de types composites : les types de tableau et les types d'enregistrement.

Constante. Voir objet.

Contrainte. Une contrainte détermine un sous-ensemble des valeurs d'un type. UN
la valeur dans ce sous-ensemble satisfait la contrainte.

Clause de contexte. Voir unité de compilation.

Déclaration. Une déclaration associe un identifiant (ou un autre
notation) avec une entité. Cette association est en vigueur dans une région de
texte intitulé la portée de la déclaration. Dans le cadre d'un 
déclaration, il existe des endroits où il est possible d'utiliser l'identifiant pour
faire référence à l’entité déclarée associée. Dans de tels endroits, l'identifiant est
dit être un simple nom de l'entité ; on dit que le nom désigne le
entité associée.

Partie déclarative. Une partie déclarative est une séquence de déclarations. Il
peut également contenir des informations connexes telles que les organes et les sous-programmes
clauses de représentation.

Dénoter. Voir déclaration.

Type dérivé. Un type dérivé est un type dont les opérations et les valeurs sont
répliques de celles d'un type existant. Le type existant est appelé le
type parent du type dérivé.

Désigner. Voir type d'accès, tâche.

Visibilité directe. Voir visibilité.

Type discret. Un type discret est un type qui possède un ensemble ordonné de
valeurs distinctes. Les types discrets sont les types énumération et entier.
Les types discrets sont utilisés pour l'indexation et l'itération, ainsi que pour les choix en cas de
déclarations et variantes d’enregistrement.

Discriminant. Un discriminant est un composant distinctif d'un objet ou
valeur d’un type d’enregistrement. Les sous-types des autres composants, voire leurs
la présence ou l'absence, peut dépendre de la valeur du discriminant.

Contrainte discriminante. Une contrainte discriminante sur un type d'enregistrement ou
Le type privé spécifie une valeur pour chaque discriminant du type.

Élaboration. L'élaboration d'une déclaration est le processus par lequel le
la déclaration produit son effet (comme la création d'un objet) ; ce processus
se produit pendant l’exécution du programme.

Entrée. Une entrée est utilisée pour la communication entre les tâches. Extérieurement, un
l'entrée est appelée de la même manière qu'un sous-programme est appelé ; son comportement interne est
spécifié par une ou plusieurs instructions d'acceptation spécifiant les actions à effectuer
effectuée lors de l'appel de l'entrée.

Type d'énumération. Un type énumération est un type discret dont les valeurs sont
représenté par des littéraux d'énumération qui sont donnés explicitement dans le type 
déclaration. Ces littéraux d'énumération sont soit des identifiants, soit
caractères littéraux.

Évaluation. L'évaluation d'une expression est le processus par lequel
la valeur de l’expression est calculée. Ce processus se produit pendant le programme
exécution.

Exception. Une exception est une situation d'erreur qui peut survenir lors
exécution du programme. Lever une exception, c'est abandonner le programme normal
exécution afin de signaler que l'erreur a eu lieu. Une exception
Le gestionnaire est une partie du texte du programme spécifiant une réponse au
exception. L'exécution d'un tel texte de programme est appelée gestion du
exception.                                           

Nom développé. Un nom développé désigne une entité déclarée
immédiatement dans une certaine construction. Un nom développé a la forme d'un
composant sélectionné : le préfixe désigne la construction (une unité de programme ; ou
un bloc, une boucle ou une instruction accept) ; le sélecteur est le simple nom de
l'entité.

Expression. Une expression définit le calcul d'une valeur.

Type à virgule fixe. Voir le type réel.

Type à virgule flottante. Voir le type réel.

Paramètre formel. Voir paramètre.

Fonction. Voir sous-programme.

Unité générique. Une unité générique est un modèle soit pour un ensemble de sous-programmes
ou pour un ensemble de forfaits. Un sous-programme ou un package créé à l'aide du
Le modèle est appelé une instance de l’unité générique. Un générique
l'instanciation est le type de déclaration qui crée une instance. UN
l'unité générique est écrite sous forme de sous-programme ou de package mais avec le
spécification préfixée par une partie formelle générique qui peut déclarer générique
paramètres formels. Un paramètre formel générique est soit un type, soit un
sous-programme ou un objet. Une unité générique est l'un des types de programmes
unité.

Gestionnaire. Voir exception.

Indice. Voir type de tableau.

Contrainte d'index. Une contrainte d'index pour un type de tableau spécifie le
limites inférieures et supérieures pour chaque plage d'index du type de tableau.

Composant indexé. Un composant indexé désigne un composant dans un tableau.
C'est une forme de nom contenant des expressions qui spécifient les valeurs du
indices du composant tableau. Un composant indexé peut également désigner un
entrée dans une famille d’entrées.

Exemple. Voir unité générique.

Type entier. Un type entier est un type discret dont les valeurs représentent
tous les nombres entiers dans une plage spécifique.

Élément lexical. Un élément lexical est un identifiant, un littéral, un
délimiteur, ou un commentaire.

Type limité. Un type limité est un type pour lequel ni affectation ni
la comparaison prédéfinie pour l'égalité est implicitement déclarée. Toutes les tâches
les types sont limités. Un type privé peut être défini comme étant limité. Un
L’opérateur d’égalité peut être explicitement déclaré pour un type limité.

Littéral. Un littéral représente une valeur littéralement, c'est-à-dire au moyen de
lettres et autres caractères. Un littéral est soit un littéral numérique, soit un
un littéral d’énumération, un littéral de caractère ou un littéral de chaîne.  

Mode. Voir paramètre.

Numéro de modèle. Un numéro de modèle est une valeur exactement représentable d'un
taper. Les opérations de type réel sont définies en termes d'opérations sur le
numéros de modèle du type. Les propriétés des numéros de modèle et de
leurs opérations sont les propriétés minimales préservées par tous
implémentations de type réel.

Nom. Un nom est une construction qui représente une entité : on dit que
le nom désigne l'entité, et que l'entité est la signification du
nom. Voir aussi déclaration, préfixe.                                             

Association nommée. Une association nommée spécifie l'association d'un
élément avec une ou plusieurs positions dans une liste, en nommant les positions.

Objet. Un objet contient une valeur. Un programme crée un objet soit en
en élaborant une déclaration d'objet ou en évaluant un allocateur. Le
déclaration ou allocateur spécifie un type pour l'objet : l'objet peut
ne contiennent que des valeurs de ce type.

Opération. Une opération est une action élémentaire associée à un ou
plus de types. Elle est soit implicitement déclarée par la déclaration du
type, ou il s’agit d’un sous-programme qui a un paramètre ou un résultat du type.

Opérateur. Un opérateur est une opération qui comporte un ou deux opérandes. UN
l'opérateur unaire est écrit avant un opérande ; un opérateur binaire s'écrit
entre deux opérandes. Cette notation est un type particulier d'appel de fonction.
Un opérateur peut être déclaré comme une fonction. De nombreux opérateurs sont implicitement
déclaré par la déclaration d'un type (par exemple, la plupart des déclarations de type
implique la déclaration de l'opérateur d'égalité pour les valeurs du type).

Surcharge. Un identifiant peut avoir plusieurs significations alternatives à la fois.
point donné dans le texte du programme : cette propriété est appelée surcharge. Pour
Par exemple, un littéral d'énumération surchargé peut être un identifiant qui
apparaît dans les définitions de deux ou plusieurs types d’énumération. L'efficace  
la signification d’un identifiant surchargé est déterminée par le contexte.
Les sous-programmes, agrégats, allocateurs et chaînes littérales peuvent également être
surchargé.

Emballer. Un package spécifie un groupe d'entités logiquement liées, telles que
en tant que types, objets de ces types et sous-programmes avec des paramètres de ceux-ci
les types. Il est écrit sous forme de déclaration de package et de corps de package. Le
La déclaration du package a une partie visible, contenant les déclarations de tous
entités qui peuvent être explicitement utilisées en dehors du package. Cela peut aussi avoir
une partie privée contenant des détails structurels qui complètent le
spécification des entités visibles, mais qui ne sont pas pertinentes pour l'utilisateur
du colis. Le corps du package contient des implémentations de sous-programmes
(et éventuellement des tâches comme d'autres packages) qui ont été spécifiées dans le
déclaration de paquet. Un package est l’un des types d’unités de programme.

Paramètre. Un paramètre est l'une des entités nommées associées à un
sous-programme, entrée ou unité générique, et utilisé pour communiquer avec le
corps de sous-programme correspondant, instruction d'acceptation ou corps générique. Un formel
Le paramètre est un identifiant utilisé pour désigner l’entité nommée dans le corps.
Un paramètre réel est l'entité particulière associée au
paramètre formel correspondant par un appel de sous-programme, un appel d'entrée ou un appel générique
instanciation. Le mode d'un paramètre formel spécifie si le
le paramètre réel associé fournit une valeur pour le paramètre formel, ou
le formel fournit une valeur pour le paramètre réel, ou les deux. Le
l'association des paramètres réels avec des paramètres formels peut être spécifiée par
associations nommées, par associations de position ou par une combinaison de
ces.

Type de parent. Voir type dérivé.

Association de position. Une association positionnelle précise l'association
d'un élément avec une position dans une liste, en utilisant la même position dans le 
texte pour spécifier l'élément.

Pragma. Un pragma transmet des informations au compilateur.

Préfixe. Un préfixe est utilisé comme première partie de certains types de noms. UN
le préfixe est soit un appel de fonction, soit un nom.

Partie privée. Voir forfait.

Type privé. Un type privé est un type dont la structure et l'ensemble de valeurs
sont clairement définis, mais ne sont pas directement accessibles à l'utilisateur du type. UN
Le type privé n'est connu que par ses discriminants (le cas échéant) et par l'ensemble des
opérations définies pour lui. Un type privé et ses opérations applicables
sont définis dans la partie visible d'un package, ou dans une partie formelle générique.
L'affectation, l'égalité et l'inégalité sont également définies pour les types privés,
sauf si le type privé est limité.                                        

Procédure. Voir sous-programme.

Programme. Un programme est composé d'un certain nombre d'unités de compilation, l'une des
qui est un sous-programme appelé programme principal. Exécution du programme
consiste en l'exécution du programme principal, qui peut invoquer des sous-programmes
déclaré dans les autres unités de compilation du programme.

Unité de programme. Une unité de programme est une unité générique, un package,
sous-programme ou unité de tâche.

Expression qualifiée. Une expression qualifiée est une expression précédée de
une indication de son type ou sous-type. Une telle qualification est utilisée lorsque, dans
son absence, l'expression peut être ambiguë (par exemple en tant que
conséquence d'une surcharge).

Lever une exception. Voir exception.

Gamme. Une plage est un ensemble contigu de valeurs de type scalaire. Une gamme est
spécifié en donnant les limites inférieure et supérieure des valeurs. Une valeur en
on dit que la gamme appartient à la gamme.

Contrainte de portée. Une contrainte de plage d'un type spécifie une plage, et
détermine ainsi le sous-ensemble des valeurs du type qui appartiennent au
gamme.

Un vrai type. Un type réel est un type dont les valeurs représentent des approximations de
les vrais chiffres. Il existe deux types de types réels : les types à virgule fixe sont
spécifié par une limite d'erreur absolue ; les types à virgule flottante sont spécifiés par un
Limite d'erreur relative exprimée sous la forme d'un nombre de chiffres décimaux significatifs.

Type d'enregistrement. Une valeur d'un type d'enregistrement est constituée de composants qui sont
généralement de différents types ou sous-types. Pour chaque composant d'un enregistrement
valeur ou un objet d'enregistrement, la définition du type d'enregistrement spécifie un
identifiant qui détermine de manière unique le composant dans l’enregistrement.

Renommer la déclaration. Une déclaration de changement de nom déclare un autre nom pour un
entité.

Rendez-vous. Un rendez-vous est l'interaction qui se produit entre deux
tâches parallèles lorsqu'une tâche a appelé une entrée de l'autre tâche, et qu'un
L'instruction d'acceptation correspondante est en cours d'exécution par l'autre tâche sur
au nom de la tâche appelante.

Clause de représentation. Une clause de représentation dirige le compilateur dans le
sélection du mappage d'un type, d'un objet ou d'une tâche sur des caractéristiques de
la machine sous-jacente qui exécute un programme. Dans certains cas,
les clauses de représentation précisent complètement le mappage ; dans d'autres cas,
ils fournissent des critères pour choisir un mappage.

Satisfaire. Voir contrainte, sous-type.

Type scalaire. Un objet ou une valeur de type scalaire n’a pas de composants.
Un type scalaire est soit un type discret, soit un type réel. Les valeurs d'un
les types scalaires sont ordonnés.  

Portée. Voir déclaration.

Composant sélectionné. Un composant sélectionné est un nom constitué d'un préfixe
et d'un identifiant appelé sélecteur. Les composants sélectionnés sont utilisés pour
désignent les composants d'enregistrement, les entrées et les objets désignés par des valeurs d'accès ;
ils sont également utilisés comme noms développés.

Sélecteur. Voir composant sélectionné.

Nom simple. Voir déclaration, nom.                                              

Déclaration. Une instruction spécifie une ou plusieurs actions à effectuer
lors de l'exécution d'un programme.

Sous-composant. Un sous-composant est soit un composant, soit un composant de
un autre sous-composant.

Sous-programme. Un sous-programme est soit une procédure, soit une fonction. Une procédure
spécifie une séquence d'actions et est invoqué par un appel de procédure
déclaration. Une fonction spécifie une séquence d'actions et renvoie également un
valeur appelée le résultat, et donc un appel de fonction est une expression. UN
Le sous-programme est écrit sous la forme d'une déclaration de sous-programme, qui spécifie son
nom, paramètres formels et (pour une fonction) son résultat ; et un sous-programme
corps qui précise la séquence d’actions. L'appel du sous-programme
spécifie les paramètres réels qui doivent être associés au formulaire formel
paramètres. Un sous-programme est l'un des types d'unités de programme.

Sous-type. Un sous-type d'un type caractérise un sous-ensemble des valeurs du
taper. Le sous-ensemble est déterminé par une contrainte sur le type. Chaque valeur dans
l'ensemble des valeurs d'un sous-type appartient au sous-type et satisfait au
contrainte déterminant le sous-type.

Sous-unité. Voir corps.

Tâche. Une tâche fonctionne en parallèle avec d'autres parties du programme. C'est
écrit sous forme de spécification de tâche (qui spécifie le nom de la tâche et
les noms et paramètres formels de ses entrées), et un corps de travail qui
définit son exécution. Une unité de tâche est l’un des types d’unités de programme. UN
le type de tâche est un type qui permet la déclaration ultérieure de n'importe quel nombre
de tâches similaires du type. Une valeur d'un type de tâche est dite désigner
une tâche.

Taper. Un type caractérise à la fois un ensemble de valeurs et un ensemble d'opérations
applicable à ces valeurs. Une définition de type est une construction de langage qui
définit un type. Un type particulier est soit un type d'accès, soit un type de tableau,
un type privé, un type d'enregistrement, un type scalaire ou un type de tâche.

Utilisez la clause. Une clause d'utilisation permet d'obtenir une visibilité directe des déclarations qui
apparaissent dans les parties visibles des packages nommés.

Variable. Voir objet.

Partie variante. Une partie variante d'un enregistrement spécifie un enregistrement alternatif
composants, en fonction d’un discriminant de l’enregistrement. Chaque valeur du
le discriminant établit une alternative particulière de la partie variante.

Visibilité. A un moment donné du texte d'un programme, la déclaration d'un
une entité avec un certain identifiant est dite visible si l'entité est un
signification acceptable pour une occurrence à ce point de l’identifiant. Le
la déclaration est visible par sélection à la place du sélecteur dans un
composant sélectionné ou à la place du nom dans une association nommée.
Sinon, la déclaration est directement visible, c'est-à-dire si l'identifiant
seul a ce sens.

Partie visible. Voir forfait.  

Avec clause. Voir unité de compilation.
